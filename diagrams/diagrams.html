On-Call Paved Road — design

Deployment-safety demo: a metric-gated canary that auto-rolls-back a bad release (v2 vs. an un-migrated severity column).

2 · The money shot — bad deploy → auto-rollback

sequenceDiagram
  participant Op as Operator (deploy v2)
  participant RO as Argo Rollout
  participant V2 as oncall v2
  participant PG as Postgres (no severity col)
  participant Prom as Prometheus
  participant AT as AnalysisRun
  Op->>RO: set image oncall=v2
  RO->>V2: canary 40%
  V2->>PG: SELECT/INSERT ... severity ...
  PG-->>V2: ERROR column "severity" does not exist
  V2-->>Prom: 500s on the canary slice
  AT->>Prom: error-rate query
  Prom-->>AT: 24% ≥ 5%
  AT->>RO: Failed → abort
  RO->>RO: 100% back to v1 (healthy)
  Note over Op,RO: ~30s · nobody paged
